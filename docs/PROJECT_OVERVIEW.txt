## 🔄 Fluxo de Funcionamento

### Cenário 1: Produção S3

```
conteúdo JSON
   ↓ (contains: directoryName = "template-landing-page")
vite.config.ts
   ↓ (reads directoryName)
base = "https://s3.../projetos/template-landing-page/"
   ↓
npm run build
   ↓
dist/ (with URLs: https://s3.../assets/index-abc123.js)
   ↓
Deploy to S3
   ↓
✅ Funciona: https://s3.../projetos/template-landing-page/
```

### Cenário 2: Iframe Admin-Pages

```
template.zip (from S3 build)
   ↓
admin-pages extracts
   ↓
Processa HTML:
  - Converte: https://s3.../assets/... → assets/...
  - Remove: directoryName = ""
   ↓
Injeta Proxy Script:
  - /assets/... → blob:http://localhost:3000/...
   ↓
React renderiza
   ↓
✅ Funciona: http://localhost:3000/landing (iframe)
```

### Cenário 3: Dev Local

```
npm run dev
   ↓
vite dev server (base = "/")
   ↓
Assets servidos em:
  - /assets/...
  - /img/...
   ↓
http://localhost:5173
   ↓
✅ Funciona: Totalmente local, sem S3
```

## 💡 Conceitos Principais

### Hook: useAssetUrl()

```
// Antes (manual)
const url = `${getBasePath()}img/header.webp`

// Depois (automático)
const getAssetUrl = useAssetUrl({ directoryName })
const url = getAssetUrl('img/header.webp')

Resultado:
  - S3 Production: https://s3.../projetos/template.../img/header.webp
  - Blob iframe: /img/header.webp (→ blob:...)
  - Dev Local: /img/header.webp
```

### Config: s3-urls.ts

```
Três funções principais:
1. isRunningInBlob()        → Detecta blob URLs
2. isProduction()           → Detecta produção
3. resolveAssetUrl(...)     → Resolve URLs dinamicamente

Uso:
  if (shouldUseAbsoluteS3Urls()) {
    // Use URLs absolutas S3
  }
```
